# 🛒 POS — ระบบขายหน้าร้าน

ระบบ Point of Sale ครบวงจร ใช้งานง่าย รองรับทั้ง **มือถือ / iPad / PC** (responsive)
สร้างด้วย **Next.js + Tailwind CSS + Supabase** พร้อม deploy บน **Vercel** ได้ทันที

## ✨ ฟีเจอร์

- **ขายหน้าร้าน (POS)** — 2 โหมดแยกกันชัดเจน: **ขายด่วน** (เลือกสินค้า → คิดเงินจบทันที เหมาะกับซื้อกลับบ้าน) และ **เปิดโต๊ะ** (กินก่อนจ่ายทีหลัง — เลือกโต๊ะ สั่งอาหารได้หลายรอบ ระบบรวมเป็นบิลเดียวของโต๊ะอัตโนมัติ **แก้ไข/ลบรายการที่สั่งไปแล้วได้ถ้ากดผิด** ค่อยเก็บเงิน/ปิดโต๊ะตอนจบมื้อ) ทั้งสองโหมดค้นหา/สแกนบาร์โค้ด, กรองตามหมวดหมู่, ส่วนลด, ชำระด้วยเงินสด/โอน/บัตร, คำนวณเงินทอนอัตโนมัติ
- **บันทึกยอดขาย** — ทุกบิลถูกบันทึกพร้อมรายการสินค้า ราคา และต้นทุน ณ เวลาขาย, ดูประวัติย้อนหลังตามช่วงวันที่, ยกเลิกบิล (คืนสต๊อกอัตโนมัติ)
- **บันทึกต้นทุน & กำไร** — กำหนดต้นทุนต่อชิ้นของสินค้า ระบบคำนวณกำไรต่อชิ้น กำไรต่อบิล และกำไรรวมให้อัตโนมัติ
- **บันทึกรายจ่าย/ต้นทุนกิจการ** — บันทึกต้นทุนเริ่มต้น/เงินลงทุน, วัตถุดิบ, ค่าเช่า, ค่าน้ำ-ค่าไฟ, เงินเดือน, ค่าขนส่ง, การตลาด ฯลฯ แยกตามหมวดหมู่ที่ปรับแต่งเองได้ ค้นหา/กรองตามหมวดหมู่/ช่วงวันที่
- **บันทึกรายได้อื่นๆ** — รายได้นอกเหนือจากยอดขาย POS เช่น เงินลงทุนเพิ่ม, รายได้ค่าบริการ, เงินคืนจากซัพพลายเออร์ — ค้นหา/กรอง/สรุปตามหมวดหมู่เหมือนหน้ารายจ่าย พร้อม **บันทึกยอดขายเองได้** สำหรับยอดขายนอกระบบ POS (เช่น ขายผ่านโทรศัพท์/แอปส่งของ) ซึ่งจะถูกนับรวมเป็นยอดขายจริงในหน้าภาพรวมและประวัติการขายทันที
- **จัดการสินค้า & สต๊อก** — เพิ่ม/แก้ไข/ลบสินค้า, หมวดหมู่, บาร์โค้ด, ตัดสต๊อกอัตโนมัติเมื่อขาย, แจ้งเตือนสินค้าใกล้หมด
- **Dashboard ภาพรวมร้าน** — ยอดขาย ต้นทุนสินค้า กำไรขั้นต้น รายได้อื่นๆ รายจ่ายอื่นๆ **กำไรสุทธิ** จำนวนบิล, กราฟยอดขายรายวัน, สินค้าขายดี Top 5, แยกยอดตามวิธีชำระเงิน, แยกรายจ่ายตามหมวดหมู่ (วันนี้ / 7 วัน / 30 วัน / เดือนนี้)
- **ระบบ role พนักงาน** — พนักงานขายและผู้จัดการเข้าถึงเท่ากัน (ขายหน้าร้าน/ประวัติขาย/จัดการสินค้า) ส่วนภาพรวมร้าน/รายได้/รายจ่าย/จัดการพนักงาน สงวนไว้ให้ **เจ้าของร้าน** เท่านั้น — บังคับสิทธิ์จริงที่ระดับฐานข้อมูล (RLS) ไม่ใช่แค่ซ่อนเมนู
- **สร้างพนักงานในแอปได้เลย + ล็อกอินด้วยเบอร์โทร** — เจ้าของร้านกด "เพิ่มพนักงาน" ตั้งเบอร์โทร+รหัสผ่านให้เอง (ไม่ต้องมีอีเมลก็สร้างได้) พนักงานใช้ **เบอร์โทรหรืออีเมล + รหัสผ่าน** ล็อกอินได้ทันที และเจ้าของร้านรีเซ็ตรหัสผ่านให้พนักงานได้ตลอดเวลา
- **Log กิจกรรม** — บันทึกทุกการเพิ่ม/แก้ไข/ลบในระบบอัตโนมัติเป็นข้อความอ่านง่าย (สินค้า, หมวดหมู่, รายรับ-รายจ่าย, โต๊ะ, พนักงาน, การขาย/ยกเลิกบิล/แก้ไขออเดอร์) พร้อมชื่อผู้ทำรายการและเวลา — หน้า "Log" เห็นได้เฉพาะเจ้าของร้าน ค้นหา/กรองตามช่วงวันที่ได้
- **ระบบล็อกอิน** — ผ่าน Supabase Auth ปลอดภัยด้วย Row Level Security

## 🧰 Tech Stack

| ส่วน | เทคโนโลยี |
|---|---|
| Frontend | Next.js 14 (App Router) + TypeScript |
| UI | Tailwind CSS (responsive: mobile / tablet / desktop) |
| Database + Auth | Supabase (PostgreSQL + Row Level Security) |
| Hosting | Vercel |

## 🚀 วิธีติดตั้ง

### 1. สร้างโปรเจกต์ Supabase

1. ไปที่ [supabase.com](https://supabase.com) → **New Project**
2. เมื่อสร้างเสร็จ ไปที่ **SQL Editor** → วางเนื้อหาทั้งหมดจากไฟล์ [`supabase/schema.sql`](supabase/schema.sql) → กด **Run**
3. ไปที่ **Authentication → Users → Add user** → สร้างบัญชีเจ้าของร้าน (อีเมล + รหัสผ่าน) — แนะนำติ๊ก *Auto Confirm User*
   - **บัญชีแรกที่สมัครจะได้สิทธิ์ "เจ้าของร้าน" อัตโนมัติ** บัญชีถัดไปจะเป็น "พนักงานขาย" โดยปริยาย — ไปปรับ role ได้ที่เมนู "พนักงาน" ในแอป (เจ้าของร้านเท่านั้นที่เห็นเมนูนี้)
4. ไปที่ **Settings → API** จด 3 ค่านี้ไว้:
   - `Project URL`
   - `anon public` key
   - `service_role` key (กด "Reveal" เพื่อดู) — **เก็บเป็นความลับ ห้ามใส่ในโค้ดฝั่งเบราว์เซอร์หรือ commit ขึ้น git เด็ดขาด** ใช้สำหรับให้เจ้าของร้านสร้างบัญชีพนักงาน/รีเซ็ตรหัสผ่านจากในแอปเท่านั้น

### 2. รันบนเครื่อง (local)

```bash
git clone https://github.com/NTPxz/POS.git
cd POS
npm install
cp .env.example .env.local   # แล้วแก้ไขใส่ค่าจาก Supabase (รวม service_role key)
npm run dev
```

เปิด http://localhost:3000 แล้วล็อกอินด้วยบัญชีที่สร้างไว้

### 3. Deploy ขึ้น Vercel

1. ไปที่ [vercel.com](https://vercel.com) → **Add New Project** → เลือก repo `NTPxz/POS`
2. ที่หน้า config ใส่ **Environment Variables**:
   - `NEXT_PUBLIC_SUPABASE_URL` = Project URL ของ Supabase
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = anon key ของ Supabase
   - `SUPABASE_SERVICE_ROLE_KEY` = service_role key ของ Supabase (**ห้ามติ๊ก "Expose to browser"** — ค่านี้ต้องอยู่ฝั่งเซิร์ฟเวอร์เท่านั้น ปกติ Vercel จะไม่ expose ให้อยู่แล้วตราบใดที่ชื่อตัวแปรไม่ขึ้นต้นด้วย `NEXT_PUBLIC_`)
3. กด **Deploy** — เสร็จแล้ว! เปิดใช้ได้จากมือถือ / iPad / PC ได้เลย

> 💡 เปิดจากมือถือแล้วกด "Add to Home Screen" จะใช้งานได้เหมือนแอปเลย

## 🗄️ โครงสร้างฐานข้อมูล

- `categories` — หมวดหมู่สินค้า
- `products` — สินค้า (ราคาขาย, ต้นทุน, สต๊อก, บาร์โค้ด)
- `dining_tables` — โต๊ะในร้าน (เจ้าของร้านตั้งค่าไว้ล่วงหน้า)
- `sales` — บิลขาย (ยอดรวม, ส่วนลด, ต้นทุนรวม, วิธีชำระ, เงินรับ/ทอน, สถานะ `open`/`completed`/`voided`, `table_id` ถ้ามาจากการเปิดโต๊ะ)
- `sale_items` — รายการสินค้าในบิล (บันทึกราคาและต้นทุน ณ เวลาขาย)
- `expense_categories` / `expenses` — หมวดหมู่และรายการรายจ่าย/ต้นทุนกิจการ
- `income_categories` / `income` — หมวดหมู่และรายการรายได้อื่นนอกเหนือจากยอดขาย POS
- `profiles` — role + เบอร์โทรของผู้ใช้แต่ละคน (`owner` / `manager` / `staff`) สร้างอัตโนมัติเมื่อสมัครผ่าน trigger, เบอร์โทร unique
- `activity_log` — log กิจกรรมทุกการเพิ่ม/แก้ไข/ลบในระบบ (เห็นได้เฉพาะ owner)
- ฟังก์ชัน `create_sale()` — บันทึกบิล "ขายด่วน" + ตัดสต๊อกใน transaction เดียว (กันข้อมูลเพี้ยน)
- ฟังก์ชัน `add_order_to_table()` — เปิดบิล "open" ใหม่ให้โต๊ะ หรือรวมรายการเข้าบิล "open" เดิมถ้ามีอยู่แล้ว (กินหลายรอบ = บิลเดียว)
- ฟังก์ชัน `update_table_order_item()` — แก้ไขจำนวน/ลบรายการที่สั่งไปแล้วในบิลที่ยังเปิดอยู่ (กันกดผิด) ปรับสต๊อก/ยอดบิลให้อัตโนมัติ
- ฟังก์ชัน `checkout_table()` — ปิดบิล "open" เป็น "completed" พร้อมส่วนลด/วิธีชำระ/เงินรับ-ทอน
- ฟังก์ชัน `void_sale()` — ยกเลิกบิล (รองรับทั้งบิล `open` และ `completed`) + คืนสต๊อก
- ฟังก์ชัน `my_role()` / `is_manager_or_above()` / `is_owner()` — ใช้ตรวจสิทธิ์ใน RLS policy ของทุกตาราง
- ฟังก์ชัน `get_email_by_phone()` — ค้นหาอีเมลจากเบอร์โทรตอนล็อกอิน (normalize ตัวเลขทั้งสองฝั่งให้ทนเว้นวรรค/ขีด)
- ฟังก์ชัน `log_action()` — เขียน log แบบข้อความอ่านง่ายจาก RPC ต่างๆ (ขาย/ยกเลิก/สั่งอาหาร/แก้ไขออเดอร์)
- trigger `log_activity()` — log อัตโนมัติสำหรับตารางที่แก้ไขตรงจากหน้าเว็บ (สินค้า, หมวดหมู่, รายรับ-รายจ่าย, โต๊ะ, พนักงาน)

### โหมด "เปิดโต๊ะ" (กินก่อนจ่ายทีหลัง) ทำงานอย่างไร

1. เจ้าของร้านตั้งจำนวน/ชื่อโต๊ะไว้ล่วงหน้าที่ปุ่ม "จัดการโต๊ะ" ในโหมดเปิดโต๊ะ
2. พนักงานแตะเลือกโต๊ะที่ลูกค้านั่ง → สั่งอาหาร → กด "ส่งออเดอร์" (เรียก `add_order_to_table`) — ทำซ้ำได้เรื่อยๆ ตลอดมื้อ ทุกรอบจะถูกรวมเป็นบิล `open` เดียวของโต๊ะนั้นอัตโนมัติ
3. กดผิด/สั่งผิดได้ — รายการ "สั่งไปแล้ว" แก้จำนวนหรือลบออกได้ทันทีถ้าบิลยังเปิดอยู่ (สต๊อก/ยอดบิลปรับให้อัตโนมัติ)
4. เมื่อลูกค้าพร้อมจ่าย กด "เก็บเงิน / ปิดโต๊ะ" → ใส่ส่วนลด/วิธีชำระ/เงินรับ → บิลเปลี่ยนเป็น `completed` โต๊ะกลับมาว่างพร้อมรับลูกค้าใหม่
5. บิล `open` (โต๊ะที่ยังไม่ปิด) จะไม่แสดงในหน้า "ประวัติการขาย" หรือนับรวมในยอดขาย/กำไรจนกว่าจะปิดโต๊ะจริง

### ล็อกอินด้วยเบอร์โทรทำงานอย่างไร

Supabase Auth ผูกบัญชีด้วยอีเมลเป็นหลัก เบอร์โทรในระบบนี้จึงเป็น "ทางลัด" ไปหาอีเมลจริงเบื้องหลัง ไม่ใช่ระบบ SMS OTP แยกต่างหาก (ไม่มีค่าใช้จ่ายส่ง SMS):

1. เจ้าของร้านสร้างพนักงานที่หน้า "พนักงาน" — ถ้าไม่กรอกอีเมล ระบบจะสร้างอีเมลปลอมให้อัตโนมัติ (เช่น `phone-0812345678@staff.pos.local`) เพื่อให้ Supabase Auth ใช้งานได้ พนักงานไม่จำเป็นต้องรู้อีเมลนี้เลย
2. ตอนล็อกอิน ถ้าพิมพ์เบอร์โทร (ไม่มี `@`) หน้า login จะเรียก `get_email_by_phone()` หาอีเมลจริงก่อน แล้วค่อย sign in ด้วยอีเมล+รหัสผ่านตามปกติ
3. บัญชีที่ไม่มีอีเมลจริงจะ **รีเซ็ตรหัสผ่านผ่านอีเมลไม่ได้** — เจ้าของร้านต้องกด "เปลี่ยนรหัสผ่าน" ให้ที่หน้า "พนักงาน" แทน

### สิทธิ์การเข้าถึงตาม role

| ตาราง/ฟีเจอร์ | staff (พนักงานขาย) | manager (ผู้จัดการ) | owner (เจ้าของร้าน) |
